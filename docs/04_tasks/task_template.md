# Task Template

Use this template for future task files under `docs/04_tasks/`.

```md
# Task: <name>

Date: `<YYYY-MM-DD>`
Status: `proposed | active | blocked | completed`

## Goal

What this task is meant to achieve.

## Why Now

Why this task matters at the current repo stage.

## Inputs

- docs to read first
- endpoints to inspect
- scripts to run

## Steps

1. concrete step
2. concrete step
3. concrete step

## Outputs

- expected doc
- expected code change
- expected verifier

## Non-Goals

What this task should not expand into.

## Completion Check

- what should be true when this task is done
```
